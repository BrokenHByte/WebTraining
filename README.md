# Учебный проект
Проект для практики навыков по созданию сервисов в фреймворке ASP.NET Core

# Сборка и запуск
1. Клонировать репозиторий с необходимой веткой
2. Для запуска необходим docker или база данных postgre.  
   Один из вариантов это запустить докер контейнер с базой из корневой папки проекта.    
   Команда: docker compose up -d  
   Схема базы данных будет создана автоматически при первом запуске приложения.  
   Если необходимо использовать локальную базу можно отредактировать строку соединения в appsettings.json (WebProject)
4. Запустить из папки проекта команду dotnet run --project WebProject
5. Можно проверить по адресу
`http://localhost:5008/swagger/index.html`

(Тесты проекта запускаются из папки проекта командой
dotnet test Tests)
(Тесты используют InMemory-провайдер в качестве замены postrgre)

# Доступное API
(Формат ответа при ошибках соответсвует Problem Details (RFC 7807))

GET /events  
**Параметры запроса**  
```
title 
(string, опциональный)
(регистронезависимый, частичное совпадение)
отбор по заголовку события

from 
(DateTime, опциональный)
отбор событий начинающихся до указанной даты

to 
(DateTime, опциональный)
фильтрация событий начинающихся до указанной даты

page
(int, опциональный, по умолчанию 1)
номер подгруппы в группе отобранных результатов

pageSize
(int, опциональный, по умолчанию 10)
количество событий на одну подгруппу 
```

```json

  "pageSize": 2, // Max = pageSize

      "title": "12345",
      "description": "12346",
      "startAt": "2026-05-13T13:30:10.871Z",
      "endAt": "2026-08-13T13:30:10.871Z"
    },
    {
      "id": "4fecc173-b4be-4fc6-96f9-04c276290099",
```

### Добавить событие
POST /events  
**Тело запроса**  
```json
{
  "title": "Заголовок",
  "description": "Описание",
  "startAt": "2026-04-27T03:11:08.393Z",
  "endAt": "2026-04-27T06:11:08.393Z"
}
```
**Возвращает**  
201 при успехе  
400 если есть нарушения в теле запроса

------------------------
### Получить событие по значению Id
GET /events/{id}  
**Пример ответа:**
```json
{
  "id": "5decc173-b4be-4fc6-96f9-04c276290099",
  "title": "Заголовок",
  "description": "Описание",
  "startAt": "2026-04-27T03:11:08.393Z",
  "endAt": "2026-04-27T06:11:08.393Z"
}
```
**Возвращает**  
200 при успехе

------------------------

**Возвращает**

**Возвращает**  
200 при успехе

### Бронирование
Порядок бронирования
1. Выбираем событие на основании **GET /events**
2. Бронируем событие через **POST /events/{id}/book**
3. Проверяем статус брони через **GET /bookings/{id}**

### Фоновый сервис обработки бронирования
**BookingBackgroundService**

Циклично
1. Каждые 100 мс проверяет есть ли забронированные события со статусом "Pending"
2. Выполняет задержку 2 секунды для каждой найденной брони переводя статус в состояние "Confirmed"

------------------------
### Забронировать событие по id 
POST /events/{id}/book

**Пример ответа:**
```json
{
  "id": "a75f87e4-de26-4a8c-93aa-9df9dd32a3b5",
  "eventId": "5decc173-b4be-4fc6-96f9-04c276290099",
  "status": "Pending"
}
```
**Возвращает**  
202 если бронь зарегистрирована 
404 если событие не найдено

------------------------
### Проверка статуса бронирования по id брони
GET /bookings/{id}

**Пример ответа:**

Возвращает полную модель бронирования
```json
{
  "id": "e1351cf3-ae1f-4882-8217-49a26e484b00",
  "eventId": "c4856d85-2955-4e1c-b173-134689f03916",
  "status": "Confirmed",
  "createdAt": "2026-05-25T06:45:49.5022756Z",
  "processedAt": "2026-05-25T06:45:51.5905483Z"
}
```
Варианты ответа:    
Pending, Confirmed, Rejected

**Возвращает**  
200 если успешно
404 если бронирование не найдено
